# 基礎功能詳細說明

本手冊詳細說明題目敘述、配置設定與測資上傳的各項功能。

---

## 一、題目敘述

此區塊用於編輯題目的完整敘述內容。

### 題目說明

撰寫題目的主要內容，說明問題背景與需求。支援 Markdown 格式。

### 輸入說明

描述程式預期接收的輸入格式，包含資料類型與範圍限制。

### 輸出說明

描述程式應輸出的格式，包含精度要求與換行規則。

### 提示

提供額外的解題提示或注意事項（選填）。

### 範例輸入/輸出

可新增多組範例供學生參考。每組範例限制 1024 字元。

---

## 二、配置設定

### 1. 支援語言

設定允許學生使用的程式語言，包含：
- C
- C++
- Python

可同時勾選多種語言。此設定會影響：
- 前端繳交介面的語言選項
- 試跑模式可接受的檔案類型
- AI 助教的判斷邏輯

### 2. 標籤管理

用於標註題目的類型或知識點，方便分類與搜尋。輸入時以逗號分隔多個標籤。

**限制**：每個標籤最多 16 字元。

### 3. 提交額度

限制每位學生針對此題目的總提交次數：
- 輸入 `-1` 代表不限次數
- 輸入 `1` 到 `500` 之間的整數代表具體次數限制

### 4. 繳交格式

規定學生繳交作業的形式：

| 格式 | 說明 | 適用情境 |
|------|------|----------|
| **程式碼** | 學生直接上傳單一原始碼檔案 | 一般題目 |
| **壓縮檔** | 學生上傳含多檔案的 ZIP | Makefile 專案 |

選擇壓縮檔時，可設定最大容量限制（1 至 1000 MB，預設 50 MB）。

## 三、測試資料

### 測資壓縮檔命名規範

系統根據檔案的前綴編號自動進行任務分組：

| 任務編號 | 檔案命名範例 | 說明 |
|----------|--------------|------|
| 任務 0 | `0000.in`, `0000.out` | 前綴 `00` |
| 任務 1 | `0100.in`, `0100.out` | 前綴 `01` |
| 任務 2 | `0200.in`, `0200.out` | 前綴 `02` |

**命名規則**：
```
[任務編號 2 位數][測資編號 2 位數].[in|out]
```

**範例**：
```
testdata.zip
├── 0000.in
├── 0000.out
├── 0001.in
├── 0001.out
├── 0100.in
├── 0100.out
└── 0101.in
└── 0101.out
```

上述結構代表：
- 任務 0：2 筆測資（0000, 0001）
- 任務 1：2 筆測資（0100, 0101）

> **注意**：輸入檔（`.in`）與輸出檔（`.out`）的數量必須一致且編號對應。

### 任務參數設定

上傳測資後，系統會自動解析並顯示各任務。可針對每個任務調整：

| 參數 | 說明 | 預設值 |
|------|------|--------|
| **時間限制** | 程式執行時間上限（毫秒） | 1000 ms |
| **記憶體限制** | 程式可用記憶體上限（MB） | 256 MB |
| **任務分數** | 該任務的配分 | 依任務數量平均分配 |

> **驗證規則**：所有任務分數總和必須等於 100 分。

## 四、試跑模式

開啟此模式後，學生可在正式提交前進行測試。

### 基礎設定

| 選項 | 說明 |
|------|------|
| **試跑次數上限** | 輸入 `-1` 不限次數，或 `1` 到 `500` |
| **顯示標準輸出** | 是否顯示 stdout 內容 |
| **顯示錯誤訊息** | 是否顯示 stderr 內容 |
| **結果可下載** | 是否允許下載執行結果 |

### 上傳公開測資

上傳僅含輸入檔（`.in`）的壓縮檔作為公開測試資料。

**命名規則**：與正式測資相同的前綴編號規則。

### 上傳正確參考程式

啟用試跑模式時，必須上傳至少一份正確的程式碼（`.c`、`.cpp` 或 `.py`）供系統產生預期輸出。

## 五、AI 助教

透過生成式 AI 模型輔助學生理解題目或除錯。

### 設定步驟

1. **開啟功能**：勾選啟用 AI 助教
2. **選擇模型**：從下拉選單選擇支援的模型
3. **選擇金鑰**：勾選課程提供的 API 金鑰
4. **上傳 AC 程式**：提供正確的參考程式碼

> **注意**：必須至少選擇一組 API 金鑰，否則無法儲存。

## 常見問題

### 提交配額設定規則？

- 輸入 `-1` 代表不限次數
- 輸入 `1` 到 `500` 之間的整數

### 什麼時候提交格式該選壓縮檔？

當題目要求學生同時提交多個原始碼或標頭檔時使用（如 Makefile 繳交）。

### 測資分數總和不是 100 怎麼辦？

調整各任務的分數欄位，確保總和為 100 分後即可提交。
