# 評測流程操作指南

本章節說明題目評測環境的進階設定，包含檔案存取權限、執行模式、靜態分析限制以及自訂評分腳本。

---

## 1. 檔案存取權限

檔案存取分為讀取與寫入兩個層級，其權限設定會影響題目資源的使用：

* **允許讀取**：開啟後程式才能執行讀取檔案的操作。若題目包含學生專用資源檔，必須開啟此權限，否則程式無法讀取資源。教師專用資源則不在此限。
* **允許寫入**：開啟後程式才能建立或修改檔案。
* **連動規則**：寫入權限必須建立在讀取權限之上。若關閉允許讀取，系統會自動強制關閉允許寫入，介面亦會顯示相關提醒。

---

## 2. 函式庫使用限制

系統透過靜態分析來限制程式能呼叫的資源，根據不同語言提供對應的限制方式：

* **語言特性**：Python 限制模組匯入，而 C 或 C++ 則限制標頭檔引用。
* **過濾模式**：每一類限制都能切換為白名單或黑名單模式。
  * **白名單**：僅允許清單內的項目。
  * **黑名單**：禁止使用清單內的項目。
  * **注意**：切換模式時會自動清空原有的清單，以避免邏輯衝突。

可設定的類別包含：語法限制、匯入限制、標頭檔限制以及函式限制。

---

## 3. 執行模式說明

系統提供三種不同的環境來因應各類題型：

| 模式名稱 | 適用情境 | 必要條件 |
| :--- | :--- | :--- |
| **一般模式** | 多數標準題目 | 無特殊要求 |
| **僅限函式** | 需特定編譯或連結方式 | 必須上傳編譯設定檔壓縮包 |
| **互動模式** | 互動式題目 | 須上傳教師端程式，可設定是否由教師端優先執行 |

> **注意**：在互動模式下，系統會鎖定並停用自訂檢查器功能。

---

## 4. 自訂檢查器與計分腳本

當標準的比對方式無法滿足需求時，可使用 Python 撰寫自訂邏輯。

### 自訂檢查器
用於程式化比對學生的輸出結果。開啟此功能時，必須上傳檢查器腳本或確認後端已有預設腳本，否則無法送出。

### 自訂計分腳本
用於定義特殊的評分規則。開啟時同樣須確保已上傳計分腳本檔案。

---

## 5. 檔案上傳限制

為了維護系統穩定，所有上傳欄位都會在瀏覽器端進行檔案大小檢查。若上傳檔案超過系統規範的限制，系統會自動清除選取項目並顯示錯誤訊息。
